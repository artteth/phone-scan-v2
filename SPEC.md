# Спецификация: Приложение инвентаризации рулонов ткани

## 1. Обзор проекта

**Название:** Fabric Roll Inventory Scanner  
**Тип:** Веб-приложение (PWA-ready)  
**Функционал:** Сканирование штрих-кодов/QR-кодов на рулонах ткани, запись измерений (метраж, усадка), управление заказами  
**Целевые пользователи:** Сотрудники склада/производства тканей

---

## 2. Структура данных

### 2.1 Заказ (Order)
```javascript
{
  id: "2020",           // Номер заказа (из штрих-кода)
  totalRolls: 5,        // Всего рулонов в заказе
  rolls: [
    {
      rollNumber: 1,    // Номер рулона (2020_1)
      factoryLength: 50,    // Заводской метраж (м)
      measuredLength: null, // Измеренный метраж (м)
      shrinkage: null,      // Усадка (%)
      status: "pending"     // pending | measured | completed
    },
    // ... остальные рулоны
  ]
}
```

### 2.2 Формат штрих-кода
- **QR-код:** `2020_1`, `2020_2`, `2020_3` и т.д.
- Где `2020` - номер заказа, `_1`, `_2` - номер рулона

## 3. UI/UX Спецификация

### 3.1 Страница сканирования (Главная)

**Элементы:**
- Заголовок "Сканер рулонов ткани"
- Кнопка "Сканировать QR-код" (активирует камеру)
- Поле ввода "Или введите вручную" (текстовое поле + кнопка)
- Список последних сканирований (до 5 штук)
- Кнопка перехода к списку всех заказов

**Поведение:**
- При сканировании: парсит код, ищет заказ, открывает форму записи
- При вводе вручную: аналогично сканированию
- Если заказ не найден: создает новый с запросом количества рулонов

### 3.2 Модальное окно записи данных рулона

**Поля:**
- Номер заказа (только чтение)
- Номер рулона (только чтение)
- Заводской метраж (числовое поле, м)
- Измеренный метраж (числовое поле, м) - опционально
- Усадка (числовое поле, %) - опционально

**Кнопки:**
- "Записать" - сохраняет данные
- "Записать частично" - сохраняет только заполненные поля

**Поведение:**
- Автоматически заполняет номер заказа и рулона из сканирования
- Показывает текущий статус рулона
- Позволяет дописывать данные порционно

### 3.3 Страница списка заказов

**Элементы:**
- Таблица/список заказов
- Для каждого заказа: номер, прогресс (X/Y рулонов), статус
- Клик по заказу - детальный вид

### 3.4 Детальный вид заказа

**Элементы:**
- Информация о заказе
- Таблица всех рулонов с их данными
- Возможность редактирования любого рулона

## 6. Google Sheets Интеграция

### 6.1 Обзор

Приложение поддерживает синхронизацию с Google Таблицей через Google Apps Script. Это позволяет:
- Хранить данные централизованно в облаке
- Синхронизировать данные между несколькими устройствами
- Сохранять данные даже после очистки локального хранилища

### 6.2 Структура Google Таблицы

Таблица "рулоны" содержит следующие колонки:

| Колонка | Описание | Тип данных |
|---------|----------|------------|
| OrderID | Номер заказа (например, "2020") | Строка |
| RollNumber | Номер рулона (1, 2, 3...) | Число |
| FactoryLength | Заводской метраж (м) | Число |
| MeasuredLength | Измеренный метраж (м) | Число |
| Shrinkage | Усадка (%) | Число |
| Status | Статус (pending/partial/completed) | Строка |
| CreatedDate | Дата создания записи | Дата |
| ModifiedDate | Дата последнего изменения | Дата |

### 6.3 Настройка Google Apps Script

1. **Создайте новую Google Таблицу**
   - Откройте https://sheets.google.com
   - Создайте новую таблицу
   - Скопируйте ID таблицы из URL (между `/d/` и `/edit`)

2. **Откройте редактор скриптов**
   - В таблице: Extensions → Apps Script
   - Или: https://script.google.com

3. **Вставьте код из файла `cod.gs`**

4. **Настройте константы в начале файла:**
   ```javascript
   const SPREADSHEET_ID = 'YOUR_SPREADSHEET_ID_HERE'; // Замените на ваш ID
   const SHEET_NAME = 'рулоны';
   ```

5. **Разверните как веб-приложение:**
   - Deploy → New deployment
   - Select type: Web app
   - Execute as: Me
   - Who has access: Anyone
   - Нажмите Deploy
   - Скопируйте URL веб-приложения

6. **Настройте приложение:**
   - Откройте настройки приложения (кнопка ⚙️)
   - Включите "Google Таблица Синхронизация"
   - Вставьте скопированный URL в поле "Google Apps Script URL"
   - Нажмите "Синхронизировать сейчас"

### 6.4 API Методы

Google Apps Script поддерживает следующие действия:

| Действие | Метод | Параметры | Описание |
|----------|-------|-----------|----------|
| `getOrders` | GET | - | Получить все заказы |
| `getOrder` | GET | orderId | Получить конкретный заказ |
| `saveRoll` | POST | orderId, rollNumber, factoryLength, measuredLength, shrinkage | Сохранить данные рулона |
| `addOrder` | POST | orderId, totalRolls | Создать новый заказ |
| `deleteRoll` | POST | orderId, rollNumber | Удалить рулон |

### 6.5 Режимы работы

- **Локальный режим** (по умолчанию): Данные сохраняются только в localStorage браузера
- **Режим синхронизации**: Данные сохраняются и в localStorage, и в Google Таблице

### 6.6 Обработка ошибок

При ошибке синхронизации:
- Данные сохраняются локально
- Пользователь получает уведомление "Данные сохранены локально (ошибка синхронизации)"
- При следующей успешной синхронизации данные будут отправлены в Google Таблицу
