## Обзор проекта

**Назначение:** Веб-приложение для учёта рулонов ткани на складе.

**Основные функции:**
- Создание заказов с указанием количества рулонов
- Учёт заводского и измеренного метража
- Расчёт усадки ткани
- Отслеживание статуса выполнения заказов
- Синхронизация между устройствами через Google Таблицы

**Целевые пользователи:** Сотрудники склада, производственные работники

## Установка и настройка

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/artteth/phone-scan-v2.git
cd phone-scan-v2/основной\ проект
```

### Шаг 2: Развёртывание на GitHub Pages

1. Зайдите на https://github.com/artteth/phone-scan-v2/settings/pages
2. В разделе **Source** выберите ветку `main` и папку `/ (root)`
3. Нажмите **Save**
4. Через 1-2 минуты сайт будет доступен по адресу:
   ```
   https://artteth.github.io/phone-scan-v2/
   ```

### Шаг 3: Настройка Google Sheets

См. подробную инструкцию в файле `GOOGLE_SHEETS_SETUP.md`

**Кратко:**
1. Создайте новую Google Таблицу
2. Скопируйте ID таблицы из URL
3. Откройте Extensions → Apps Script
4. Вставьте код из `cod.gs`
5. Замените `SPREADSHEET_ID` на ваш ID
6. Разверните как веб-приложение (Deploy → New deployment)
7. Скопируйте URL веб-приложения

### Шаг 4: Включение синхронизации

1. Откройте приложение в браузере
2. Нажмите **⚙️ Настройки**
3. Включите **"Google Таблица Синхронизация"**
4. Вставьте URL Google Apps Script
5. Нажмите **"🔄 Синхронизировать сейчас"**

## Настройка Google Sheets

### Структура таблицы

Таблица автоматически создаётся при первой синхронизации.

**Название листа:** `рулоны`

**Колонки:**

| № | Название | Тип | Описание |
|---|----------|-----|----------|
| A | OrderID | Строка | Номер заказа (например, "2024") |
| B | RollNumber | Число | Номер рулона (1, 2, 3...) |
| C | FactoryLength | Число | Заводской метраж (м) |
| D | MeasuredLength | Число | Измеренный метраж (м) |
| E | Shrinkage | Число | Усадка (%) |
| F | Status | Строка | Статус (pending/partial/completed) |
| G | CreatedDate | Дата | Дата создания записи |
| H | ModifiedDate | Дата | Дата изменения записи |

### Пример данных

| OrderID | RollNumber | FactoryLength | MeasuredLength | Shrinkage | Status | CreatedDate | ModifiedDate |
|---------|------------|---------------|----------------|-----------|--------|-------------|--------------|
| 2024 | 1 | 50 | 48.5 | 3.0 | completed | 2024-01-01 | 2024-01-02 |
| 2024 | 2 | 52 | | | pending | 2024-01-01 | 2024-01-01 |

## Использование

### Создание заказа

1. Нажмите **"📋 Все заказы"**
2. Нажмите **"+ Новый заказ"**
3. Введите номер заказа (например, `2024`)
4. Укажите количество рулонов (например, `20`)
5. Нажмите **"Создать заказ"**

### Добавление данных рулона

1. Откройте заказ из списка
2. Нажмите **"Изменить"** на нужном рулоне
3. Введите:
   - **Заводской метраж** (обязательно)
   - **Измеренный метраж** (по желанию)
   - **Усадка** (по желанию)
4. Нажмите **"Записать"** или **"Записать частично"**

### Статусы рулонов

| Статус | Описание | Цвет |
|--------|----------|------|
| **Ожидает** | Данные не внесены | Жёлтый |
| **Частично** | Внесены не все данные | Синий |
| **Готов** | Все данные внесены | Зелёный |

### Статусы заказов

| Статус | Описание |
|--------|----------|
| **Ожидает** | Ни один рулон не начат |
| **В работе** | Есть рулоны в работе или готовые |
| **Завершён** | Все рулоны готовы |

## Синхронизация данных

### Как работает

1. **При загрузке страницы:**
   - Данные загружаются из Google Таблицы
   - Сохраняются в localStorage браузера

2. **Авто-синхронизация:**
   - Каждые **30 секунд** данные обновляются
   - Только если вкладка активна (видима)

3. **При возврате на вкладку:**
   - Данные автоматически обновляются

4. **При сохранении данных:**
   - Данные отправляются в Google Таблицу
   - Сохраняются локально

### Режимы работы

| Режим | Описание |
|-------|----------|
| **Онлайн** | Данные синхронизируются с Google Таблицей |
| **Оффлайн** | Данные сохраняются только локально |

### Индикатор синхронизации

В **правом верхнем углу** отображается маленький индикатор 🔄 во время синхронизации:

- **Появляется** при начале синхронизации
- **Крутящийся спиннер** — данные загружаются
- **Исчезает** когда синхронизация завершена
- **Не мешает работе** — маленький, в углу

### Настройки синхронизации

Откройте **⚙️ Настройки**:

- **Включить синхронизацию** — переключатель режима
- **Google Apps Script URL** — адрес скрипта
- **🔄 Синхронизировать сейчас** — принудительная синхронизация
- **🗑️ Очистить локальные данные** — сброс кэша браузера

## Восстановление проекта

### Если что-то сломалось

#### 1. Проверьте консоль браузера

Нажмите **F12** → вкладка **Console**

Ищите ошибки:
- `Uncaught ReferenceError` — переменная не определена
- `SyntaxError` — ошибка в JavaScript
- `Failed to fetch` — проблема с синхронизацией

#### 2. Очистите кэш

**Mac:** `Cmd+Shift+R`  
**Windows:** `Ctrl+Shift+R`

Или через DevTools:
1. Откройте DevTools (F12)
2. Нажмите правой кнопкой на кнопке обновления
3. Выберите **"Очистить кэш и жесткая перезагрузка"**

#### 3. Восстановите из рабочего коммита

```bash
cd "/path/to/phone-scan-v2/основной проект"
git checkout <commit-hash> -- index.html
git commit -m "Restore from <commit-hash>"
git push
```

**Рабочие коммиты:**
- `d3b5c35` — Чистая версия без ошибок
- `9efc979` — Исправлены все ошибки
- `4be1a9e` — Добавлена авто-синхронизация (текущий)

#### 4. Проверьте Google Apps Script

1. Откройте https://script.google.com
2. Выберите ваш проект
3. Проверьте, что код из `cod.gs` загружен
4. Проверьте развёртывание:
   - **Deploy** → **Manage deployments**
   - **Who has access:** Anyone
   - **Execute as:** Me

### Проверка работоспособности

1. ✅ Откройте приложение в браузере
2. ✅ Все кнопки нажимаются
3. ✅ Можно создать новый заказ
4. ✅ Можно добавить данные рулона
5. ✅ Данные синхронизируются (зелёное уведомление)
6. ✅ В Google Таблице появляются записи
